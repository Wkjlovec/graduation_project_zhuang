表 5-1 操作描述列原始 HTTP 方法对照

| 编号 | 测试场景 | 原操作描述（HTTP 方法+路径） |
|:---:|---------|--------------------------|
| 1 | 获取分区列表 | GET /api/forum/sections |
| 2 | 注册并登录两个用户 | POST /api/auth/register × 2 |
| 3 | 发帖→评论→搜索全链路 | POST /api/forum/posts → POST /api/forum/posts/{postId}/comments → GET /api/search/posts?keyword=... |
| 4 | 通知首页与媒体首页 | GET /api/notifications/home、GET /api/media/home |
| 5 | 个人通知与标记已读 | GET /api/notifications/my → POST /api/notifications/{notificationId}/read |
| 6 | 无 token 写操作拦截 | POST /api/forum/posts（不携带 Authorization 头） |
| 7 | 无权限操作拦截 | DELETE /api/forum/posts/{postId}（用户 B 的令牌删除用户 A 的帖子） |
| 8 | 并发重复点赞 | POST /api/forum/posts/{postId}/like × 2（双线程同时发起） |
| 9 | 登出后会话失效 | POST /api/auth/logout → GET /api/users/me（携带旧令牌） |
| 10 | 令牌刷新流程 | POST /api/auth/refresh（携带 refreshToken） |
| 11 | 令牌过期拦截 | GET /api/users/me（等待 accessToken 过期后发起） |
