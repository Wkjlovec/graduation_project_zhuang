# 毕设可交付范围（降级版）

本文档用于约束后续开发复杂度，保证“稳定可运行优先 + 工作量可答辩”。

## 1. 必做（Must Have）

1. 基础设施：Docker + Nacos + MySQL + Redis  
2. 网关路由 + 注册发现  
3. 基础鉴权：
   - 登录/注册
   - Access Token + Refresh Token
   - 登出后会话失效
   - 会话隔离防串号
4. 论坛主流程：
   - 发帖、列表、详情、评论、点赞
   - 帖子/评论基础权限控制
5. 双端接入：
   - PC 端可完成登录、发帖、评论、浏览
   - 移动端可完成登录、发帖、评论、浏览
   - 首页常驻通知、搜索、音乐推荐、书籍推荐板块
6. 基础缓存与演示数据：
   - 帖子列表/详情缓存
   - 初始化测试数据
7. 扩展服务（轻量版）：
   - notification-service（公告与个人通知）
   - search-service（帖子关键词搜索）
   - media-service（音乐/书籍推荐）

## 2. 选做（Nice to Have）

- 自动刷新令牌前端拦截器
- 收藏、私信、通知模板管理
- 搜索排序优化（高亮、权重）

## 3. 本阶段不做（Won't Have）

- 大规模攻击防护（WAF、风控平台、复杂反爬）
- 分布式事务复杂治理
- 高可用集群级容灾演练

## 4. 设计原则

1. 先保证端到端可运行，再做增强。  
2. 能用配置解决，不引入重型中间件。  
3. 能用单服务落地，不额外拆服务。  
4. 安全做“基础可交付”，避免过度工程化。
