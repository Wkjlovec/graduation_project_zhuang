第五章 系统测试

5.1 测试环境

本系统的测试在本地开发环境中完成，硬件和软件配置如下。

服务器端硬件配置：CPU 为 Intel Core i5 系列，主频 2.4GHz 以上，内存 16GB，硬盘 512GB SSD。操作系统为 macOS 或 Linux（Ubuntu 22.04），Java 运行环境为 JDK 17，Maven 版本为 3.9.x。

基础设施组件通过 Docker Compose 编排部署在同一台机器上：MySQL 8.4（字符集 utf8mb4，端口 3306）、Redis 7.4（开启 AOF 持久化，端口 6379）、Nacos 2.3.2（standalone 模式，端口 8848）。三个容器运行在名为 forum-net 的 bridge 网络中。

后端各服务的监听端口分别为：网关 8080、认证服务 8081、论坛服务 8082、通知服务 8083、媒体服务 8084、搜索服务 8085。前端 PC 端运行在 5173 端口，移动端运行在 5174 端口。

前端测试环境：Node.js v22，npm 包管理器。测试浏览器为 Chrome 130+、Firefox 132+、Edge 130+，移动端使用 Android 手机浏览器进行验证。

性能测试分为缓存基准测试和并发负载测试两个部分，分别验证缓存机制的有效性和系统在不同并发规模下的承载能力。

5.7 测试结论

功能测试覆盖了系统全部 22 个 API 接口和 11 个端到端业务场景，测试通过率 100%，系统在正常流程和异常路径上的行为均符合需求分析阶段定义的预期。安全测试的 9 个用例全部通过，系统在认证拦截、权限隔离、并发去重和基础输入防护四个维度达到了设计目标。性能测试验证了 Redis 缓存机制对帖子列表接口响应时间的改善效果，并发负载测试在 100 并发用户规模下验证了系统的承载能力。兼容性测试确认了系统在主流桌面浏览器和移动端浏览器上的正常运行。测试过程中发现的缺陷均已修复并通过回归验证。当前测试基于本地开发环境执行，未进行云服务器部署下的大规模压力测试，后续如部署到生产环境可进一步补充。
