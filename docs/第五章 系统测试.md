第五章 系统测试

5.1 测试环境

全部测试在本地开发环境中完成，后端服务、基础设施和前端应用部署在同一台机器上。

硬件为 Intel Core i5 系列处理器（主频 2.4GHz 以上），16GB 内存，512GB SSD。操作系统为 macOS 或 Linux（Ubuntu 22.04），后端运行在 JDK 17 上，构建工具为 Maven 3.9.x。

基础设施由 Docker Compose 编排：MySQL 8.4（utf8mb4 字符集，端口 3306）、Redis 7.4（AOF 持久化，端口 6379）、Nacos 2.3.2（standalone 模式，端口 8848），三个容器运行在 forum-net bridge 网络中。后端各服务的端口与 4.5 节一致——网关 8080、认证 8081、论坛 8082、通知 8083、媒体 8084、搜索 8085。前端 PC 端和移动端分别运行在 5173 和 5174 端口，测试浏览器为 Chrome 130+。

5.2 功能测试

功能测试采用基于 Python 脚本驱动的全链路黑盒测试方法。测试脚本为 deploy/scripts/run_system_regression.py，该脚本通过标准 HTTP 请求按照预设的业务流程依次调用系统各接口，将实际返回的状态码和业务响应码与预期值进行比对，全部通过后自动生成 Markdown 格式的回归测试报告。

测试共覆盖 11 个场景，其中场景 1 至 5 验证系统的正常业务流程，场景 6 至 9 验证系统在异常输入和越权操作下的防护行为，场景 10 和 11 验证令牌生命周期管理机制。测试用例与结果如表 5-1 所示。

5.3 性能测试

性能测试分为缓存基准测试和并发负载测试两个部分，分别验证缓存机制的有效性和系统在不同并发规模下的承载能力。

缓存基准测试使用 deploy/scripts/benchmark_forum_cache.py 脚本执行。该脚本对帖子列表接口（GET /api/forum/posts?page=0&size=10）先执行一次冷启动请求（缓存未命中，穿透到 MySQL 数据库查询），随后执行 30 次热缓存请求（缓存命中，直接从 Redis 返回），分别记录响应时间并计算性能改善幅度。测试结果如表 5-3 所示。

5.4 测试结论

功能测试覆盖了系统全部 22 个 API 接口和 11 个端到端业务场景，测试通过率 100%。系统在正常业务流程和异常路径（包括无令牌访问、越权操作、并发去重和会话失效）上的行为均符合第三章需求分析阶段定义的预期。性能测试验证了 Redis 缓存机制对帖子列表接口响应时间的改善效果，并发负载测试在 10 至 100 并发用户规模下验证了系统的基本承载能力。当前测试基于本地开发环境执行，尚未进行云服务器部署下的大规模压力测试、基于专业工具的安全漏洞扫描以及单元测试覆盖率统计，这些将作为后续改进方向在第六章中提出。
