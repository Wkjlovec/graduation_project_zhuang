## 摘要

论坛系统在高校师生的课程答疑、学习资料共享和校园信息传播中承担着重要角色。现有校园论坛普遍存在单体架构扩展性不足、前后端耦合导致维护困难、缺乏移动端适配以及测试验证手段不完善等问题，制约了系统在高校场景中的持续发展。

本文设计并实现了一个基于微服务架构的高校论坛系统。后端采用 Spring Boot 3 与 Spring Cloud Alibaba 构建，通过 Nacos 实现服务注册与配置管理，经由 Spring Cloud Gateway 统一路由与鉴权转发，将系统拆分为认证服务、论坛服务、通知服务、搜索服务和媒体服务五个独立模块。数据持久化采用 MySQL，缓存与会话管理采用 Redis，用户认证基于 JWT 结合 Spring Security 实现登录、令牌刷新与登出的完整闭环。前端基于 Vue 3 分别采用 Element Plus 和 Vant 构建 PC 端与移动端应用，实现帖子发布与浏览、分区浏览、二级评论、点赞、通知已读标记和分页筛选等核心功能。系统配套 Docker Compose 基础设施编排、一键启停与健康检查脚本，以及 Python 编写的功能测试、缓存基准测试和并发负载测试脚本，形成可自动重跑、结果可对照的验证链路。测试结果表明，功能测试覆盖 11 个端到端业务场景和全部 22 个 API 接口，通过率为 100%；缓存机制有效提升了接口响应速度；并发负载测试在 10 至 100 并发用户规模下验证了系统的基本承载能力，能够满足高校师生日常交流与学习协作的实际需求。

**关键词：** 微服务架构；Spring Boot；Vue 3；JWT；Redis；高校论坛

## Abstract

Forum systems play an important role in course Q&A, study material sharing, and campus information dissemination among university teachers and students. Existing campus forums commonly suffer from limited scalability of monolithic architectures, high maintenance costs caused by front-back coupling, lack of mobile adaptation, and insufficient testing and validation mechanisms.

This thesis designs and implements a microservice-based forum system for university scenarios. The backend is built with Spring Boot 3 and Spring Cloud Alibaba, using Nacos for service registration and configuration management, and Spring Cloud Gateway for unified routing and authentication forwarding. The system is decomposed into five independently deployable modules: authentication, forum, notification, search, and media services. MySQL is used for data persistence and Redis for caching and session management. User authentication is implemented through JWT combined with Spring Security, covering the complete cycle of login, token refresh, and logout. The frontend is developed with Vue 3, using Element Plus for the PC client and Vant for the mobile client, implementing core functions such as post publishing and browsing, section-based browsing, two-level comments, likes, notification read-state marking, and paginated filtering. The system is accompanied by Docker Compose infrastructure orchestration, one-click startup and health-check scripts, and Python-based functional testing, cache benchmarking, and concurrent load testing scripts, forming a reproducible and comparable validation pipeline. Test results indicate that functional tests cover 11 end-to-end business scenarios and all 22 API endpoints with a 100% pass rate; the caching mechanism effectively improves API response times; and concurrent load tests under 10 to 100 simultaneous users verify the system's basic capacity, meeting the practical needs of daily communication and learning collaboration among university users.

**Keywords:** microservice architecture; Spring Boot; Vue 3; JWT; Redis; university forum
