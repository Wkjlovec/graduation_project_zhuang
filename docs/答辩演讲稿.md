## 第二部分：背景与创新点（约 40 秒）

[PPT-2 过渡] [PPT-3]

高校师生对论坛有持续需求，但现有校园论坛有三个明显问题：单体架构耦合导致扩展困难，缺乏移动端适配，功能测试依赖手工操作。

本文的改进体现在三个方面：一是用 Spring Cloud 微服务架构做了完整的服务拆分，二是设计了 JWT 加 Redis 的混合认证方案支持令牌主动撤销，三是配了三套 Python 自动化测试脚本。

---

## 第三部分：需求与技术路线（约 30 秒）

[PPT-4 过渡] [PPT-5]

系统有游客和注册用户两种角色，功能拆成认证、帖子、评论、点赞、通知、搜索、媒体和网关八个模块。非功能性需求覆盖性能、安全、可扩展、可维护和兼容五个维度。技术路线是需求分析、架构设计、分服务编码，最后自动化测试验证。

---

## 第四部分：架构与数据库（约 1 分 20 秒）

[PPT-6 过渡] [PPT-7]

系统分四层：用户接入层是 PC 端和移动端两套 Vue 前端，网关层用 Spring Cloud Gateway 统一路由和 JWT 鉴权，业务服务层有五个微服务全部注册到 Nacos，基础设施层是 MySQL、Redis 和 Nacos 通过 Docker Compose 编排。

选微服务而不是单体，是因为单体改一个模块就要全局重新部署。三个服务共享一个 MySQL 是当前体量下的务实折中。

[PPT-8]

数据库六张表、五个实体、七条关系，点赞是多对多通过关联表拆解。全部 22 个 RESTful 接口按模块分布在五个服务中，网关用白名单区分公开和受保护接口。

---

## 第五部分：核心实现与测试（约 1 分 20 秒 + 2 分钟视频）

[PPT-9 过渡] [PPT-10]

认证方面，纯 JWT 无法主动撤销，所以加了 Redis 存储会话——登出时删掉 Redis 记录，令牌立即失效。缓存方面，帖子列表缓存 2 分钟，写操作自动淘汰缓存，冷启动 59 毫秒降到热缓存 15 毫秒。并发点赞用应用层 exists 加数据库唯一约束做双重去重，零锁开销。

[PPT-13 过渡] [PPT-14]
