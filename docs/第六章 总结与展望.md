第六章 总结与展望

6.1 总结

随着高校信息化进程的持续推进，在线论坛系统在课程答疑、学习资料共享和校园信息传播中扮演着愈发重要的角色。本文针对现有校园论坛普遍存在的单体架构扩展性不足、前后端耦合、缺乏移动端适配以及测试验证手段不完善等问题，设计并实现了一个基于微服务架构的高校论坛系统。全文的主要工作和成果可概括为以下几个方面。

（1）完成了系统的需求分析与架构设计。通过对高校论坛使用场景的调研，将系统用户划分为游客和注册用户两种角色，梳理了用户认证、帖子管理、评论互动、点赞、通知服务、搜索服务、媒体推荐和网关访问控制八个功能模块的用例关系，并从性能、安全、可扩展性、可维护性和兼容性五个维度明确了非功能性需求。系统采用前后端分离的 B/S 架构，后端基于 Spring Boot 3 与 Spring Cloud Alibaba 构建，将业务逻辑拆分为网关、认证、论坛、通知、搜索和媒体六个独立服务，通过 Nacos 实现服务注册与配置管理，经由 Spring Cloud Gateway 统一路由与鉴权。

（2）完成了数据库设计与核心业务的实现。数据库包含 user_account、forum_section、forum_post、post_comment、post_like 和 notification_message 共 6 张业务表，表结构满足第三范式，forum_post 中 author_name 和 section_name 两个冗余字段属于有意为之的反范式设计，用以避免帖子列表查询时的多表 JOIN。用户认证基于 JWT 结合 Spring Security 实现，配合 Redis 存储会话状态，形成了登录、令牌刷新与登出的完整闭环。网关层的 JwtRelayFilter 通过 JWT 签名校验和 Redis 会话检查双重验证，实现了对公开接口和受保护接口的统一访问控制。论坛核心业务实现了帖子的发布、编辑、删除和分页查询，评论的二级结构和权限校验，以及基于联合唯一约束的点赞去重机制。Redis 缓存策略对帖子列表和详情分别设置了 2 分钟和 5 分钟的过期时间，写操作通过注解淘汰相关缓存，保证了数据的最终一致性。

（3）完成了前端双端应用的开发。基于 Vue 3 分别采用 Element Plus 和 Vant 构建了 PC 端与移动端应用，两端共享统一的 API 调用层和 Pinia 状态管理逻辑，在界面交互上各自适配对应终端的使用习惯。前端响应拦截器实现了 401 状态码的静默刷新机制，通过等待队列防止多个并发请求同时触发令牌刷新导致的竞争问题。

（4）搭建了容器化基础设施与自动化测试体系。MySQL、Redis 和 Nacos 通过 Docker Compose 编排部署，Bash 脚本负责后端服务的顺序启动、健康探测和有序停止。Python 脚本承担全链路回归测试和缓存基准测试任务，回归测试覆盖 11 个端到端业务场景和全部 22 个 API 接口，通过率均为 100%。缓存基准测试和并发负载测试分别验证了 Redis 缓存对接口响应时间的改善效果以及系统在 10 至 100 并发用户下的基本承载能力。

6.2 存在的问题与展望

受开发周期和个人能力的限制，系统在功能完整性、性能优化和测试覆盖等环节仍存在改进空间。以下列举目前的不足及后续改进方向。

（1）搜索服务的检索能力有待增强。当前的搜索服务采用内存中逐条遍历和字符串 contains 匹配的方式实现关键词过滤，这种方案在数据量较小时能够满足需求，但随着帖子数量增长，匹配效率将显著下降，且不支持分词、模糊匹配和相关度排序等高级检索特性。后续可考虑引入 Elasticsearch 全文搜索引擎，通过倒排索引和分词分析器提升检索精度和响应速度。

（2）系统尚未实现完善的日志采集与监控机制。当前各微服务仅依赖 Spring Boot 默认的控制台日志输出，缺少统一的日志收集、存储和查询平台。在生产环境中，分布式系统的故障定位往往需要跨多个服务节点关联日志信息。后续可引入 ELK（Elasticsearch + Logstash + Kibana）日志采集栈，结合链路追踪工具实现请求全链路的可观测性。

（3）测试覆盖范围有待进一步扩展。当前测试工作以功能回归测试和缓存性能测试为主，尚未进行基于专业工具的安全漏洞扫描（如 SQL 注入、XSS 攻击检测）、多浏览器和多设备的兼容性测试，以及基于 JUnit 的后端单元测试覆盖率统计。后续可引入 OWASP ZAP 进行自动化安全扫描，使用 Selenium 或 Playwright 实现前端跨浏览器自动化测试，并建立持续集成流程以保证每次代码变更的测试覆盖率不低于预设阈值。

（4）系统的部署架构尚未进行云环境下的生产级验证。当前测试和开发均在本地环境完成，单机部署无法体现微服务架构在水平扩展和负载均衡方面的优势。后续可将系统部署至云服务器，借助 Kubernetes 实现各服务的弹性伸缩和滚动更新，并接入 CDN 加速前端静态资源的分发，以评估系统在真实网络环境和更大用户规模下的表现。

（5）前端的交互体验和功能丰富度有进一步提升的空间。当前系统提供了论坛基本的发帖、评论、点赞和搜索功能，但在内容富文本编辑、图片上传、用户头像、个人主页、私信、收藏、热门排行等方面尚未涉及。后续可根据实际用户反馈逐步引入这些功能模块，进一步提升论坛的社区活跃度和用户粘性。
