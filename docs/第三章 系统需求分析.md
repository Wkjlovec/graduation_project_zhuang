第三章 系统需求分析

## 3.1 系统概述

本文设计的微服务论坛系统面向高校场景，为在校师生提供帖子发布与浏览、分区讨论、评论互动、信息检索和通知触达等在线交流功能。系统的用户分为两种状态：未登录的游客可以浏览帖子列表、公告摘要、搜索结果和媒体推荐等公开内容；完成注册并登录的用户则可以进一步使用发帖、评论、点赞和查看个人通知等需要身份认证的功能。系统不区分学生、教师等角色，所有注册用户享有相同的操作权限，帖子和评论的编辑删除权限通过判断当前用户是否为内容作者来控制。分区和公告等基础数据通过系统启动时的种子数据初始化机制完成配置。

系统采用前后端分离的 B/S 架构，用户通过浏览器访问 PC 端或移动端页面，所有请求经由统一网关路由至后端各微服务处理。后端按业务职责拆分为认证服务、论坛服务、通知服务、搜索服务和媒体服务五个独立模块，各模块通过 Nacos 注册中心进行服务发现，彼此之间以 RESTful 接口通信，数据持久化由 MySQL 完成，热点数据缓存和会话状态由 Redis 管理。系统的整体业务流程为：用户注册并登录后获取访问令牌，携带令牌即可访问论坛的全部功能；网关层根据请求路径和方法判断是否需要认证，GET 类的公开浏览接口允许匿名访问，而发帖、评论、点赞等写操作接口则要求携带有效令牌。系统总体架构如图 3-1 所示。
