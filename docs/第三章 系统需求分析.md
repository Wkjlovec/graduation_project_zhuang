第三章 系统需求分析

## 3.1 系统概述

本文设计的微服务论坛系统面向高校场景，为在校师生提供帖子发布与浏览、分区讨论、评论互动、信息检索和通知触达等在线交流功能。系统的目标用户可划分为三类：学生是系统的主要使用群体，通过论坛发布学习笔记、提出课程问题、分享经验心得；教师作为内容参与者，可在对应分区下发布课程相关的通知或参与答疑讨论；管理员负责系统的后台维护，包括分区管理、公告发布和数据初始化等运维操作。

系统采用前后端分离的 B/S 架构，用户通过浏览器访问 PC 端或移动端页面，所有请求经由统一网关路由至后端各微服务处理。后端按业务职责拆分为认证服务、论坛服务、通知服务、搜索服务和媒体服务五个独立模块，各模块通过 Nacos 注册中心进行服务发现，彼此之间以 RESTful 接口通信，数据持久化由 MySQL 完成，热点数据缓存和会话状态由 Redis 管理。系统的整体业务流程为：用户注册并登录后获取访问令牌，携带令牌访问论坛的各项功能；未登录的游客可浏览帖子列表、公告摘要、搜索结果和媒体推荐等公开内容，但发帖、评论、点赞和查看个人通知等操作需要完成身份认证。系统总体架构如图 3-1 所示。
